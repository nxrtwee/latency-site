# Latency — сайт

Установочный сайт медиаплеера **Latency**: лендинг, загрузки, документация и changelog.
Чистый **HTML + CSS + JS**, без сборки и зависимостей. Двуязычный (RU / EN).

## Структура

```
latency site/
├── index.html          лендинг + загрузки + галерея скриншотов
├── docs/index.html     документация → чистый URL /docs (установка, SoundCloud, Discord, FAQ…)
├── changelog/index.html  история версий → чистый URL /changelog (таймлайн)
├── .nojekyll           чтобы GitHub Pages не трогал папки
└── assets/
    ├── css/styles.css   дизайн-система (тёмная тема, неоновый акцент)
    ├── js/main.js       i18n RU/EN, меню, определение платформы, лайтбокс
    └── img/
        ├── icon.png         логотип приложения
        ├── shots/           реальные скриншоты десктопа (hero + галерея)
        └── mobile/          скриншоты мобильной версии (iOS/Android)
```

## Локальный просмотр

Сайт статический — просто откройте `index.html` в браузере. Либо поднимите
локальный сервер (чтобы пути работали как на проде):

```bash
python -m http.server 8080
# затем http://localhost:8080
```

## Скриншоты

Реальные скриншоты лежат в `assets/img/shots/` (десктоп) и
`assets/img/mobile/` (телефон). Чтобы заменить — положите файл с тем же именем.
Десктопные снимки используются с натуральными пропорциями (≈16:9), мобильные —
вертикальные (9:19.5). Изображения не сжаты; при желании прогоните их через
[squoosh.app](https://squoosh.app/) или `cwebp` для меньшего веса.

## Деплой

Сайт портативный и работает на любом статическом хостинге.

### GitHub Pages
1. Залейте содержимое папки в репозиторий (отдельный `latency-site` или ветку `gh-pages`).
2. **Settings → Pages → Source** → нужная ветка, папка `/ (root)`.
3. Файл `.nojekyll` уже добавлен — ассеты отдадутся корректно.
4. Свой домен — через `Settings → Pages → Custom domain` (+ файл `CNAME`).

### Vercel / Netlify
Импортируйте репозиторий. Сборка не нужна:
- **Build command** — пусто;
- **Output directory** — корень (`.`).

## Обновление под новый релиз плеера

При выходе новой версии Latency поправьте:
- номер версии и имена файлов в секции `#download` (`index.html`);
- добавьте блок версии в `changelog/index.html`;
- ссылки на загрузку ведут на `releases/latest` — их менять не нужно.
